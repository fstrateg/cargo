1. При добавлении груза поставить два поле дату с: по: (актуальность заявку по дате). По закрытию периода заявка автоматически закрывается и переходит в архив заявок.
2. Сделать счётчик просмотров (заходов) по заявке.
3. Добавить к имени пользователя в скобках его айди. Sfinx (id128788).
4. Каждая заявка имеет свой уникальный код. Отображать скорей всего в названии. Или отдельное поле.
5. Сделать поиск по номеру заявки и по айди пользователя. В первом случае открывается заявка, во-втором карточка пользователя.
6. Разместив заявку по маршруту, мы сразу получаем список водителей, кто готов ехать по указанному маршруту. Водитель, заранее настраивает свой список предпочтительных поездок. Меняет его по необходимости. Каждый водитель имеет свою карточку (страницу). Где грузоотправитель смотрит его данные: ФИО, телефон, срок участия в системе, наличие авто, отзывы, номер в топе, количество выполненных перевозок.
7. Водитель, нажав «Готов ехать» в заявке оповещает (письмо, смс, сообщение в приложение, сообщение на сайте) грузоотправителя о его заинтересованности грузом. При этом конечно есть возможность позвонить по указанному номеру.
8. Добавить в профиле, поля для загрузки сканов. Паспорт, трудовой договор, карточку предприятия для ТОО и ИП. Для частника паспорт.
9. В профиле добавить вкладку Авто (подвижной состав). Там водители заполняю Номер головы, номер прицепа, марку, тип. Загружаю сканы тех паспортов авто.
10. Каждая заявка, должна иметь логическое завершение. Удалена или закрыта по выполнению. Закрытием служит внесение данных по полю исполнитель (айди), сумма, оценка, примечания. Ряд этих полей надо сделать обязательных. Потом на основании этих данных сделаем отчеты.
